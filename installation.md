# VoWERP3 — Installation & Deployment Guide

## Prerequisites

| Component | Version |
|-----------|---------|
| Node.js | 18+ (recommended: 22 LTS) |
| pnpm | 9+ |
| Python | 3.12+ |
| MySQL | 8.0+ |

---

## 1. Frontend (vowerp3ui)

### Local Development

```bash
# Clone the repo
git clone https://github.com/sidheshsarda/vowerp3ui.git
cd vowerp3ui

# Install pnpm (if not installed)
npm install -g pnpm

# Install dependencies
pnpm install

# Create environment file
cp .env .env.local
```

Edit `.env.local`:
```env
USE_NEXT_PROXY=true
NEXT_PUBLIC_BACKENDHOST=http://localhost:8000
NEXT_PUBLIC_API_BASE_URL=/api
```

```bash
# Run dev server (port 3000)
pnpm dev
```

Access at `http://localhost:3000` or `http://{subdomain}.localhost:3000`

### Production Build & Deploy

```bash
# Build
pnpm build

# Test the production build locally
pnpm start
```

#### Files to copy to the server

Copy the **entire project folder** to the server, or at minimum these:

```
vowerp3ui/
├── .next/              # Build output (generated by pnpm build)
├── node_modules/       # Dependencies (or run pnpm install on server)
├── public/             # Static assets
├── package.json
├── pnpm-lock.yaml
├── next.config.ts
├── .env                # Environment config (create on server)
└── .env.local          # Local overrides (create on server)
```

#### On the server

```bash
cd vowerp3ui

# Option A: Install dependencies on server (if node_modules not copied)
npm install -g pnpm
pnpm install --frozen-lockfile

# Option B: If you copied node_modules, skip install

# Create .env for production
cat > .env << 'EOF'
USE_NEXT_PROXY=false
NEXT_PUBLIC_BACKENDHOST=http://<BACKEND_IP>:8000
NEXT_PUBLIC_API_BASE_URL=/api
EOF

# Build on the server (if .next/ not copied)
pnpm build

# Start production server
pnpm start
# Runs on port 3000 by default

# To use a custom port:
PORT=3001 pnpm start
```

#### Run as a background service (using PM2)

```bash
# Install PM2
npm install -g pm2

# Start the app
pm2 start pnpm --name "vowerp3ui" -- start

# Auto-restart on server reboot
pm2 startup
pm2 save

# View logs
pm2 logs vowerp3ui

# Restart after new deployment
pm2 restart vowerp3ui
```

## 2. Backend (vowerp3be) — Compiled Executable (No Source Code)

The backend is compiled into a standalone `.exe` using PyInstaller. No Python installation needed on the server.

### Deployment Package

**File:** `E:\sjm\deploy\vowerp3be-deploy.zip` (8.4 MB)

### On the server

1. **Extract** `vowerp3be-deploy.zip` to `C:\vowerp3be\` (or any path)

The extracted folder structure:
```
vowerp3be/
├── vowerp3be.exe          # The compiled server executable
├── env/
│   ├── database.env       # Database connection config
│   └── keys.env           # JWT secret keys
└── _internal/             # PyInstaller runtime (do not modify)
```

2. **Edit `env/database.env`** with production database credentials:
```env
DATABASE_USER=<mysql_user>
DATABASE_PASSWORD=<mysql_password>
DATABASE_HOST=<mysql_host>
DATABASE_PORT=3306
DATABASE_DEFAULT=vowconsole3
ENV=production
```

3. **Edit `env/keys.env`** with production secrets:
```env
SECRET_KEY=<your-secret-key>
ACCESS_TOKEN_EXPIRE_MINUTES=30
REFRESH_TOKEN_EXPIRE_DAYS=7
```

4. **Run the server:**
```cmd
cd C:\vowerp3be
vowerp3be.exe
```
The API server starts on port 8000.

5. **Run as a Windows Service (optional):**

Using [NSSM](https://nssm.cc/) (Non-Sucking Service Manager):
```cmd
nssm install VoWERP3Backend C:\vowerp3be\vowerp3be.exe
nssm set VoWERP3Backend AppDirectory C:\vowerp3be
nssm start VoWERP3Backend
```

### Building from Source (Development)

If you need to rebuild from source:

```bash
# Clone the repo
git clone https://github.com/sidheshsarda/vowerp3be.git
cd vowerp3be

# Create virtual environment
python -m venv .venv

# Activate (Windows)
.venv\Scripts\activate

# Activate (Linux/Mac)
source .venv/bin/activate

# Install dependencies
pip install -r requirements.txt
```

#### Configure database

Create/edit `env/database.env`:
```env
DATABASE_USER=<mysql_user>
DATABASE_PASSWORD=<mysql_password>
DATABASE_HOST=<mysql_host>
DATABASE_PORT=3306
DATABASE_DEFAULT=vowconsole3
ENV=development
```

Create/edit `env/keys.env`:
```env
SECRET_KEY=<your-secret-key>
ACCESS_TOKEN_EXPIRE_MINUTES=30
REFRESH_TOKEN_EXPIRE_DAYS=7
```

```bash
# Run dev server
uvicorn src.main:app --reload --host 0.0.0.0 --port 8000
```

### Production Deployment

```bash
cd vowerp3be

# Create virtual environment
python -m venv .venv
source .venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Set environment to production in env/database.env
# ENV=production

# Run with uvicorn (production)
uvicorn src.main:app --host 0.0.0.0 --port 8000 --workers 4
```

#### Run as a background service (using PM2)

```bash
pm2 start "uvicorn src.main:app --host 0.0.0.0 --port 8000 --workers 4" --name "vowerp3be"
pm2 save
```

#### Or using systemd (Linux)

Create `/etc/systemd/system/vowerp3be.service`:
```ini
[Unit]
Description=VoWERP3 Backend
After=network.target

[Service]
User=<your-user>
WorkingDirectory=/path/to/vowerp3be
ExecStart=/path/to/vowerp3be/.venv/bin/uvicorn src.main:app --host 0.0.0.0 --port 8000 --workers 4
Restart=always

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable vowerp3be
sudo systemctl start vowerp3be
```

## 3. Docker Deployment (Alternative)

### Frontend

```bash
cd vowerp3ui

docker run -d -p 3000:3000 --name vowerp3ui vowerp3ui
```

### Backend

```bash
cd vowerp3be
docker build -t vowerp3be .
docker run -d -p 8000:8000 --env-file env/database.env --name vowerp3be vowerp3be
```

## 4. Nginx Reverse Proxy (Recommended for Production)

To handle subdomain routing (e.g., `sls.vowerp.co.in`, `dev3.vowerp.co.in`):

```nginx
server {
    listen 80;
    server_name *.vowerp.co.in vowerp.co.in;

    # Frontend
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    # Backend API
    location /api/ {
        proxy_pass http://localhost:8000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Host $host;
    }
}
```

## 5. Quick Reference

| Service | Port | Start Command |
|---------|------|---------------|
| Frontend (dev) | 3000 | `pnpm dev` |
| Frontend (prod) | 3000 | `pnpm start` |
| Backend (dev) | 8000 | `uvicorn src.main:app --reload --host 0.0.0.0 --port 8000` |
| Backend (prod) | 8000 | `uvicorn src.main:app --host 0.0.0.0 --port 8000 --workers 4` |

### Environment Variable Summary

| Variable | Where | Purpose |
|----------|-------|---------|
| `USE_NEXT_PROXY` | Frontend `.env` | `true` for local dev (proxy API through Next.js), `false` for production |
| `NEXT_PUBLIC_BACKENDHOST` | Frontend `.env` | Backend URL (e.g., `http://localhost:8000`) |
| `NEXT_PUBLIC_API_BASE_URL` | Frontend `.env` | API base path (always `/api`) |
| `DATABASE_USER` | Backend `env/database.env` | MySQL username |
| `DATABASE_PASSWORD` | Backend `env/database.env` | MySQL password |
| `DATABASE_HOST` | Backend `env/database.env` | MySQL host |
| `DATABASE_PORT` | Backend `env/database.env` | MySQL port (default: 3306) |
| `DATABASE_DEFAULT` | Backend `env/database.env` | Default database name (`vowconsole3`) |
| `ENV` | Backend `env/database.env` | `development` or `production` |
| `SECRET_KEY` | Backend `env/keys.env` | JWT signing key |
